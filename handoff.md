# nefor-dash — Handoff

Pixel-perfect вёрстка дашбордов Альфа-Бизнеса, мигрированная со статики на **Next.js 15**.
Цель проекта — точная вёрстка и «живые» анимации; при миграции всё сохранено 1:1.

## Стек

- **Next.js 15.5** (App Router) · **React 19** · **TypeScript 5.9**
- Без UI-библиотек. Глобальный CSS (перенесён вербатим из исходной статики).
- Шрифты — локальные woff2 (`Alfa Interface Sans`, `Styrene UI`).
- Деплой — **Vercel** из ветки `main` (фреймворк определяется автоматически).

## Запуск

```bash
npm install
npm run dev        # dev-сервер (по умолчанию :3000; в проекте использовался -p 8734)
npm run build      # прод-сборка (проверка типов). НЕ запускать при живом dev — см. «Подводные камни»
npm start          # прод-сервер после build
```

## Роуты

| URL         | Что это |
|-------------|---------|
| `/`         | Главная. Внутри — переключаемые варианты **v1** и **v2** (тоггл в панели tools, НЕ отдельный роут) |
| `/current`  | Новый макет дашборда («Бухгалтер») |

Роута `/2` больше нет — v2 стал внутристраничным вариантом `/` (это форк v1: контент у них байт-в-байт одинаковый, различается только сайдбар).

## Как перенесена вёрстка (важно!)

Крупная статичная разметка (сайдбары, контент, Spotlight, твикер) перенесена **байт-в-байт**
из оригинальных HTML через `dangerouslySetInnerHTML`, а не переписана вручную в JSX. Строки
лежат в `components/dashboard/htmlPartials.ts`, сгенерированы скриптом
`scratchpad/extract.py` (нормализует пути ассетов `../assets|assets → /assets`). Это
гарантировало пиксель-перфект и целостность анимаций. Поведение навешивается типизированными
хуками, которые опрашивают DOM (`document.querySelector`) в `useEffect` с полным cleanup —
как в исходных vanilla-скриптах.

> Если правите статичную разметку — правьте оригиналы и перегенерируйте `htmlPartials.ts`,
> либо (проще для мелочей) редактируйте строки в `htmlPartials.ts` напрямую. Со временем
> имеет смысл поэтапно переписать партиалы в настоящий JSX.

## CSS

- Все 6 файлов в `styles/` перенесены **вербатим** (правились только `url()` → `/assets`,`/fonts`).
- `menu2.css` грузится **глобально** (в layout): его правила сайдбара скоуплены под `.v2`/`.m2-*`
  и НЕ задевают v1/current; секция твикера — общая инфраструктура панели.
- `styles/tools.css` — только НОВОЕ (карточки-селектор, сегмент v1/v2, рамка-обрезка).

## Панель tools

Живёт в **root layout** (`ToolsProvider` → `ToolsPanel`), поэтому **переживает смену роута**.

- **Селектор дашбордов** (карточки «Предприниматель» = Главная, «Бухгалтер» = Current) —
  клик делает `router.push` (клиентская навигация): контент и тело панели подменяются,
  **панель не закрывается** (бесшовный своп).
- **Тоггл v1/v2** (только на Главной) — меняет `variant` в контексте, без смены роута.
- **Параметры v2** — 6 свитчей стека счетов; двигают сайдбар через контекст → `useV2Page`.
- Открывается/закрывается FAB; **закрывается по Esc**.
- **Рамка:** при открытии вьюпорт дашборда ОБРЕЗАЕТСЯ через `clip-path` (содержимое 1:1, НЕ
  масштабируется; лишнее по краям отсекается), вокруг — чёрный фон на `html`. Отступы 12px,
  справа 332 (панель 320 + зазор 12).

Состояние контекста: `dashboard` (из pathname), `variant`, `v2State` (6 булей), `panelOpen`.

## Neuro-модуль

`NeuroBar` (умная строка снизу) на v2 и `/current`. Клик → разворот пилюли (магнит к
курсору), Enter/лупа → фуллскрин-чат с «мышлением» и blur-стримингом захардкоженного ответа.
Esc-цепочка: чат → Spotlight → сворачивание строки. Весь моушен — порт 1:1 из `js/neuro-bar.js`.

## Подводные камни

- **StrictMode (dev)** дважды прогоняет эффекты — во всех хуках сделан строгий cleanup
  (снятие слушателей, `clearTimeout/Interval`, `cancelAnimationFrame`). `tasksFinal` в хуках
  задач захардкожен, иначе второй прогон читает уже очищенный DOM.
- **`React.memo` на MainV1/MainV2 обязателен** — иначе апдейт контекста перерисовывает их и
  `dangerouslySetInnerHTML` пере-инжектит статику, стирая JS-выставленные стили (top'ы стека).
- **Не запускать `next build` при живом `next dev`** — общий `.next` ломается
  (`Cannot find module './694.js'`); лечится `rm -rf .next` + рестарт dev.
- **Fast Refresh** ругается на смену числа хуков «на горячую» — помогает полный рестарт dev.
- **Оверлеи** (`.nbar/.spot/.chat/.twk`, `.board`) — вне трансформируемого контента (у fixed
  внутри `transform`-предка ломается позиционирование); neuro-обёртка — `display:contents`.
